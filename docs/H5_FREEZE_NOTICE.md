# H5 冻结说明

本说明自 `2026-04-29` 生效。

结论：
- AIRS 前端后续只继续开发和维护响应式主站。
- 独立 H5 进入 `legacy / frozen` 状态，暂时保留，不删除。
- 本轮不改路由行为、不改部署配置、不切换 `m.airsindex.com`，只做标记和说明。

## H5 后续仍需修的问题

只修阻断问题，不修体验型问题。

允许继续修复：
- 白屏、启动失败、资源加载失败。
- 搜索不可用、搜索后无法进入职业详情。
- 首页、职业详情页、护城河全景图、分享图等核心页面无法打开。
- `m.airsindex.com` 或旧 H5 链接无法访问。
- 会导致核心链路完全中断的严重数据或脚本错误。

默认不修：
- 视觉风格落后于主站。
- 动画、排版、微交互不一致。
- 新模块没有同步到 H5。
- 非阻断的细节体验问题。

## 不再同步到 H5 的内容

以下内容后续只在响应式主站迭代，不再回灌到 H5：
- 新模块、新页面、新入口。
- 首页新交互、新版内容编排、新视觉优化。
- 搜索体验增强，但不包括“完全不可搜索”的阻断修复。
- 职业详情页的新卡片、新解释层、新图形表现。
- 护城河全景图的新筛选、新交互、新视觉表达。
- 分享图的新模板、新样式、新传播文案能力。
- 主站主题、编辑器、运营能力、埋点优化等非阻断增强。

## 对原则 6 的执行口径

当前响应式主站代码已经覆盖以下核心能力：
- 首页展示。
- 搜索与职业跳转。
- 职业详情页。
- 护城河全景图。
- 职业分享图生成与分享动作。

因此，从产品开发策略上，响应式主站即日起视为移动端唯一主版本。

补充说明：
- `m.airsindex.com` 现在仍保留旧 H5 可访问性。
- 在真正切换域名前，仍建议补做一轮移动浏览器真机与线上回归，确认首页、搜索、详情、护城河图、分享图链路稳定。

## m.airsindex.com 后续迁移建议

建议分三步执行：

1. 保持当前状态
   - 继续保留 `m.airsindex.com` 与旧 `/m/...` 链接可访问。
   - 不删除旧 H5，不改当前兼容跳转。

2. 以响应式主站作为验收基线
   - 以后所有新功能只在响应式主站开发。
   - 以移动浏览器对响应式主站的首页、搜索、详情、护城河图、分享图作为上线验收基线。

3. 稳定后再切域名
   - 待响应式主站移动端稳定后，将 `m.airsindex.com` 重定向到主站移动端入口。
   - 重定向时保留必要的 `soc`、`lang`、`entry` 等参数映射，确保旧详情链接可落到主站详情页。
   - 先灰度、后全量，观察一段时间再决定是否彻底下线独立 H5。
